# General Campaign Feedback

- When editing a campaign creative variant, the save changes button is getting squished when the accordion panels are opened. The save button should maintain it's height.
- Saving creates a new version of the creative, but it doesn't actually trigger the call to the orchestrator, which if the user hits fix layout action, it triggers the orch call and generates a new version as well. This make it so that it stores two versions with basically the same changes. Either the save button generates the version entirely or the we just call it fix layout/regenerate.
- In the creative editor, the color palette is not persisted after hitting save/fix layout. It should be saved between versions and sent in the prompt to the orchestrator.
- We can remove the logo panel from the creative editor.
- When reviewing the the campaign variant/creatives prompts, we should make it so that the input is editable right way instead of having the user git the edit raw prompt dropdown.
- If the user already generated a brief for a campaign, then selects new variants/creatives (i.e.: 1:1 linkedIn, 9:16 ig at first, then selects 1:1 facebook), the header, description and cta fields are empty. They should be filled if user selects more. Another way we can look at it is in the PromptComposer add a new pill button to allow the user to select which social variants they want to create and in the review it's not editable. Keep in mind that we also use the PromptComposer in the photoshoot, which is a bit different than campaigns.
