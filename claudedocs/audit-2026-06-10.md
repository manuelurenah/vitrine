# Vitrine UI Audit — 2026-06-10

Comprehensive read-only review of `src/components/**`, `src/app/**`, and design tokens (`src/app/globals.css`) against the Web Interface Guidelines and the `impeccable:frontend-design` anti-pattern reference.

Scope: 83 UI files. Foundations (tokens, fonts, layout, primitives) read in full; representative views sampled for each domain (auth, onboarding, campaigns, photoshoot, catalog, brand, assets). Cross-codebase greps used to size systemic issues.

> This is a documentation pass. No code changes were made. Fix commands are listed per finding.

---

## Anti-Patterns Verdict

**Fail — leans AI-generated, but with real craft underneath.**

The design tokens are unusually strong: a real OKLCH-adjacent palette, articulated bloom shadows, opsz-aware font axes, fluid type scale, a serious motion vocabulary. Someone designed this. But the *composition* falls into the 2024-2025 AI-tells repeatedly:

| Tell | Where |
|---|---|
| 🚩 **AI palette: dark-mode-first w/ neon volt-to-ion accent** | tokens default `color-scheme: dark`, `--volt: #00ff9d`, `--ion: #19f0ff`, "bloom-volt" halos everywhere |
| 🚩 **Gradient text on every hero** | `Wordmark.tsx:15`, `LoginPitch.tsx:17`, `WelcomeStep.tsx:35` — three different "v"/"vitrine"/"shot on demand" all use `from-volt to-ion bg-clip-text text-transparent` |
| 🚩 **Glassmorphism used decoratively** | 27 `backdrop-blur` uses; `AuthCard.tsx:9` is the textbook example (`bg-bg-1/70 backdrop-blur-md`) |
| 🚩 **Identical card grids** | `WelcomeStep` 3-card icon+heading+text trio, `PresetGrid` 8 same-sized preset tiles, `DnaStep` 7 same-sized DnaCards, `CatalogGrid` product card matrix — 42 instances of `rounded-[14|18]` cards |
| 🚩 **Nested cards** | `InputStep.tsx:146,209` — `bg-bg-2` `<article>` wraps `bg-bg-1` `<Card>`s |
| 🚩 **Center-aligned everything** | 35 `text-center` + 21 `mx-auto` — onboarding, login, campaigns landing, catalog empty state all use the same centered-column composition |
| 🚩 **Mono-as-shorthand for "technical"** | `font-mono text-[10.5px] uppercase tracking-[0.12em]` is the eyebrow / metadata recipe in **everything** — appears 80+ times |
| 🚩 **Icon-in-pill above every heading** | `WelcomeStep`, `ProcessingStep`, `InputStep`, `EmptyState` (assets), `OnboardingFrame` step-dots — soft-glowing volt-bordered circles with a 28-36px Dna/Sparkles/Upload icon |
| 🚩 **`//` comment syntax as eyebrow** | `// step 1 of 3`, `// welcome back`, `// catalog`, `// {campaign.brief.goal}` — used as decorative noise rather than meaningful metadata |

What lifts it above pure slop: typography hierarchy is competent (Bricolage + Space Grotesk + JetBrains Mono is distinctive enough), the buzz/cooking statuses have personality, and `CampaignWizard` shows real interaction design thinking (URL-based step state, debounced previews, brand-layer disclosure). But a stranger would call AI on the first glance.

**Top 5 most critical issues:**
1. **`Chip` primitive renders `<span>` but is used as an interactive control with `onClick`** (Chip.tsx:24, PhotoshootWizard.tsx:564, AssetUploader.tsx:498-502) — not keyboard-accessible, not screen-reader-accessible.
2. **TopBar search "input" is a non-interactive `<div>` masquerading as a real control** (TopBar.tsx:44) — visible ⌘k label sets an expectation that doesn't exist; users will tab past it.
3. **Onboarding InputStep form inputs have no `<label>` association** (InputStep.tsx:169,222,254-263; DnaStep.tsx:126-131,231-237) — `<h4>` card titles are not labels; screen readers will announce inputs as unlabeled.
4. **`<a href="#">` placeholder links shipped to production** (AuthCard.tsx:36,40; EmailDisclosure.tsx:64) — terms/privacy/forgot-password navigate to current page, breaking trust.
5. **Hardcoded `left-[232px]` couples PhotoshootWizard footer to Sidebar width** (PhotoshootWizard.tsx:560,723) — any sidebar layout change silently breaks the action bar.

**Overall quality (0-100):** ~62. Strong foundation, careless application. The token system is doing 80% of the work; the composition undermines it.

**Recommended next steps:**
1. Run `/normalize` to map inline hex/rgba to tokens and fix the Chip semantic bug (highest-leverage fix).
2. Run `/harden` for the form labeling + dead-link cleanup.
3. Run `/distill` on the centered-card-grid views (Welcome, Catalog empty, DnaStep) before adding more.

## Detailed Findings by Severity

### 🔴 Critical

**C1 · Chip used as button but renders `<span>`**
- **Location**: `src/components/ui/Chip.tsx:24` (primitive), `src/components/photoshoot/PhotoshootWizard.tsx:564` (ratio selector), `src/components/assets/AssetUploader.tsx:498-502`, `src/components/catalog/CatalogGrid.tsx:31-33`
- **Category**: Accessibility
- **Description**: `Chip` accepts `HTMLAttributes<HTMLSpanElement>` and renders `<span>`. PhotoshootWizard wires `onClick={() => setRatio(r)}` onto it; CatalogGrid uses Chip as a filter without onClick (it's purely decorative). Either way, a click target on a `<span>` is not in the tab order, has no implicit `role="button"`, and has no Enter/Space activation.
- **Impact**: Keyboard users cannot select ratios in PhotoshootWizard. Screen readers announce nothing actionable.
- **WCAG**: 2.1.1 Keyboard (A), 4.1.2 Name/Role/Value (A)
- **Recommendation**: Either (a) ship a `ChipButton` variant that renders `<button>` and migrate interactive uses, or (b) make `Chip` polymorphic with an `as` prop. CatalogGrid's "all/live/archived" chips need real filter behavior or should be downgraded to `<span>` Badge-style.
- **Command**: `/normalize` (consolidate Chip variants), then `/harden` (verify keyboard paths).

**C2 · TopBar search is a non-interactive `<div>`**
- **Location**: `src/components/shell/TopBar.tsx:44-48`
- **Category**: Accessibility, UX
- **Description**: Renders a `<div>` styled like an input, with placeholder text "search" and a "⌘k" shortcut hint. There is no handler, no input, no keyboard shortcut. The element is `min-w-[240px]` and visually dominant.
- **Impact**: Users tab past it (not focusable); when they click, nothing happens; mouse users discover a dead element. Worse, the ⌘k label promises a global command palette that doesn't exist.
- **Recommendation**: Either implement the command palette (the most user-respectful fix) or remove the affordance until it ships. If keeping as placeholder, at minimum render `<button disabled>` with `aria-disabled="true"` and remove the ⌘k hint.
- **Command**: `/clarify` (remove broken promise) or feature work.

**C3 · Onboarding inputs lack `<label>` association**
- **Location**: `src/components/onboarding/InputStep.tsx:169-180` (URL), `:222-227` (brand name), `:280-283` (description Textarea), `:230` (logo via label — OK); `src/components/onboarding/DnaStep.tsx:126-131` (identity), `:231-237` (tagline), `:248-255` (description)
- **Category**: Accessibility
- **Description**: `CardHead` renders an `<h4>` for the section title; the `<Input>` below has no `id`/`htmlFor`/`aria-label`/`aria-labelledby`. `DnaCard` does the same with `<span className="t-eyebrow">`. The visual relationship is clear; the programmatic one is not.
- **Impact**: Screen readers announce these as "edit, blank" with no field name. Voice control users cannot say "click brand name".
- **WCAG**: 1.3.1 Info & Relationships (A), 3.3.2 Labels or Instructions (A), 4.1.2 (A)
- **Recommendation**: Wire `useId()` in each Card/DnaCard so `CardHead` renders the title inside a `<label htmlFor={id}>` and passes the same id to its child. CampaignWizard (`CampaignWizard.tsx:704-717`) already does this correctly — use it as the reference.
- **Command**: `/harden`.

**C4 · Dead `<a href="#">` placeholder links**
- **Location**: `src/components/login/AuthCard.tsx:36,40` (terms, privacy); `src/components/login/EmailDisclosure.tsx:64` (forgot password); `src/components/login/EmailDisclosure.tsx` form has `action="#"`.
- **Category**: UX, trust, accessibility (some screen readers announce empty fragments).
- **Description**: Pre-auth screens reference real legal/auth pages that don't exist. Clicking either reloads the login page (URL becomes `/#`), which on a session-error redirect can look like a broken loop.
- **Impact**: First-touch users hit a dead end on the most legally-load-bearing copy on the site.
- **Recommendation**: Decide (a) ship `/terms`, `/privacy`, password-reset routes, or (b) replace the affordances with non-link text ("by continuing you agree to our terms and privacy policy — coming soon") and disable the password input until SSO-only is the real story.
- **Command**: `/clarify` then routing work.

**C5 · Sidebar width hardcoded twice into PhotoshootWizard footer**
- **Location**: `src/components/photoshoot/PhotoshootWizard.tsx:560`, `:723`
- **Category**: Architecture, Responsive
- **Description**: `fixed bottom-0 left-[232px] right-0` magic-numbers the Sidebar's `w-[232px]` (`Sidebar.tsx:61`). Three places to update if the sidebar ever changes. Mobile is not handled (the sidebar is always rendered; the layout grid `grid-cols-[232px_1fr]` in `Shell.tsx:15` has no breakpoint).
- **Impact**: Silent layout breakage on any sidebar change; today the wizard footer covers content on narrow viewports because the whole app assumes ≥232+content.
- **Recommendation**: Either (a) extract the sidebar width to a CSS custom property (`--sidebar-w: 232px`) consumed by both, or (b) re-parent the action bar inside the main column instead of `position: fixed`.
- **Command**: `/extract` (token), then `/adapt` (mobile layout).

### 🟠 High

**H1 · Gradient text on three hero headings**
- **Location**: `Wordmark.tsx:15`, `LoginPitch.tsx:17`, `WelcomeStep.tsx:35`
- **Category**: Anti-pattern (AI tell)
- **Impact**: Flagged in `frontend-design` as decorative-rather-than-meaningful. Hits the user three times in the first 30 seconds of the experience.
- **Recommendation**: Keep gradient on `Wordmark` (it's a logo — meaning is identity). Drop it from the body headings; let the type weight + Bricolage opsz do the work.
- **Command**: `/quieter`.

**H2 · Glassmorphism on auth card**
- **Location**: `src/components/login/AuthCard.tsx:9` (`bg-bg-1/70 backdrop-blur-md`)
- **Category**: Anti-pattern
- **Impact**: First impression. The radial bloom behind it is the brand moment; the glass weakens it. 27 total `backdrop-blur` uses across the codebase suggests this is a reflex, not a decision.
- **Recommendation**: Drop the blur. Let the card sit cleanly over the bloom. Audit other blurs — keep them only on overlays that actually need to reduce content underneath (Modal scrim is fine).
- **Command**: `/distill`.

**H3 · "Icon-in-pill above heading" template repeats**
- **Location**: `WelcomeStep.tsx:29`, `ProcessingStep.tsx:120`, `InputStep.tsx:148`, `assets/AssetsGallery.tsx:136`, `AssetUploader.tsx:352`, `AddProductForm.tsx:368`
- **Category**: Anti-pattern (templated)
- **Impact**: Six screens, same composition: 56-96px volt-bordered circle, icon centered, eyebrow + h-tag below. Looks AI-generated because it *is* a template.
- **Recommendation**: Pick one or two of these where the icon truly anchors meaning (DNA → Dna icon on onboarding works); make the rest typographic.
- **Command**: `/distill` or `/bolder`.

**H4 · Repeated "identical card grids"**
- **Location**: `WelcomeStep.tsx:45-65` (3 cards), `PresetGrid.tsx:71-110` (8 tiles), `DnaStep.tsx:124-256` (7 DnaCards), `CatalogGrid.tsx:49-86` (n product cards), `EditProductForm.tsx`/`AddProductForm.tsx` thumbnail grids
- **Category**: Anti-pattern
- **Recommendation**: At least one of these can absorb hierarchy (e.g., DnaStep "business overview" already gets `md:col-span-2`, expand the pattern: highlight one card per row as the priority). PresetGrid is the worst offender — 8 same-sized cards convey no hierarchy between "always-on" presets and the rest.
- **Command**: `/arrange`.

**H5 · Nested cards in InputStep**
- **Location**: `src/components/onboarding/InputStep.tsx:146,209` (outer `<article>` `bg-bg-2`) wraps `:354-365` `<Card>` `bg-bg-1`
- **Category**: Anti-pattern
- **Impact**: Visual noise; the inner cards have their own border + bg + padding that fight the outer article's framing.
- **Recommendation**: Flatten — let the outer article be the only container; inner sections separated by spacing + eyebrow titles, no second border.
- **Command**: `/distill`.

**H6 · "Top up" button in Sidebar is dead + uses hex `#0a0a0f`**
- **Location**: `src/components/shell/Sidebar.tsx:89-94`
- **Category**: UX, theming
- **Description**: No `onClick`/`href`; clicking is a no-op. Text color is `text-[#0a0a0f]` instead of `text-fg-on-volt` — the token already exists for this purpose.
- **Impact**: User with a low Buzz balance clicks "top up" expecting to top up. Nothing happens.
- **Recommendation**: Link to `https://civitai.com/purchase/buzz` (same URL used in `CampaignWizard.tsx:990`). Replace `#0a0a0f` with `var(--fg-on-volt)` / `text-fg-on-volt`. In light theme `--fg-on-volt` is `#ffffff`, so the hex actively breaks light mode.
- **Command**: `/normalize` for the color, `/clarify` for behavior.

**H7 · `MoreHorizontal` IconButton in TopBar has no handler**
- **Location**: `src/components/shell/TopBar.tsx:50`
- **Category**: UX
- **Description**: Renders a "more" button with no onClick. Tab-focusable, click-able, does nothing.
- **Recommendation**: Wire it (likely user menu / settings) or remove it.
- **Command**: `/clarify` or feature work.

**H8 · ProcessingStep status changes not announced**
- **Location**: `src/components/onboarding/ProcessingStep.tsx:133` (`<ul>`)
- **Category**: Accessibility
- **Description**: The task list flips between pending → running → done as the scrape progresses, but there's no `aria-live` region. Screen reader users get no audio update.
- **Recommendation**: Wrap with `role="status" aria-live="polite"`. The currently-running label (`:128-130`) should also be aria-live polite so changes are announced.
- **Command**: `/harden`.

**H9 · DnaStep readiness bar lacks progressbar ARIA**
- **Location**: `src/components/onboarding/DnaStep.tsx:270-276`
- **Category**: Accessibility
- **Description**: Width-driven progress bar with no `role="progressbar"`, `aria-valuemin`, `aria-valuemax`, `aria-valuenow`.
- **Recommendation**: Add the four attributes; expose the percentage as `aria-valuenow={readiness}`.
- **Command**: `/harden`.

**H10 · `<img>` not optimized via `next/image`**
- **Location**: `CatalogGrid.tsx:59`, `assets/AssetsGallery.tsx:113`, `assets/AssetDetailView.tsx:416`, `EditProductForm.tsx` (similar pattern), `CreativeCard.tsx:365,469`
- **Category**: Performance
- **Description**: Six `<img>` uses, all `eslint-disable @next/next/no-img-element`'d. Two of them are generated images (CreativeCard) where the URL is dynamic and remote — `next/image` would also need remotePatterns config. Catalog/Assets/Asset detail thumbnails are the ones with actual perf upside.
- **Impact**: No automatic responsive `srcset`; no AVIF; cumulative layout shift risk on grids that load N at once.
- **Recommendation**: Convert thumbnail grids to `next/image fill` with `sizes`. Keep `<img>` for generated content (where width/height aren't known ahead of time).
- **Command**: `/optimize`.

**H11 · `transition-[width]` on upload progress bars**
- **Location**: `AddProductForm.tsx:576`, `EditProductForm.tsx:509`, `AssetUploader.tsx:603`
- **Category**: Performance, anti-pattern
- **Description**: Animating `width` triggers layout. For a tiny progress bar this is fine in isolation; in a list of uploads it can compound.
- **Recommendation**: Use `transform: scaleX()` from a `transform-origin: left` anchor.
- **Command**: `/optimize`.

### 🟡 Medium

**M1 · Hardcoded `rgba(0,255,157,0.18)` for the volt halo**
- **Location**: `AssetsGallery.tsx:140`, `AssetUploader.tsx:353`, `AddProductForm.tsx:368`
- **Category**: Theming
- **Description**: `--volt-soft: rgba(0, 255, 157, 0.12)` exists; these inline at 0.18 — different value, same intent. Light theme will not switch because the value is inline.
- **Recommendation**: Add `--volt-soft-strong` if 0.18 is the desired token, or just use `var(--volt-soft)`.
- **Command**: `/normalize`.

**M2 · Step dots in OnboardingFrame have no progress ARIA**
- **Location**: `src/components/onboarding/OnboardingFrame.tsx:32-41`
- **Category**: Accessibility
- **Description**: Three colored pills indicating onboarding progress, no semantic association. CampaignWizard's `StepDots` uses `role="tablist"` (which is also wrong — they're not tabs) but at least exposes step index.
- **Recommendation**: Use `role="list"` + `<li>` with `aria-current="step"` on the active one, plus `aria-label="step 2 of 3"` on the container.
- **Command**: `/harden`.

**M3 · CampaignWizard `StepDots` mis-uses `role="tablist"`**
- **Location**: `src/components/campaigns/CampaignWizard.tsx:367-394`
- **Category**: Accessibility
- **Description**: `role="tab"` on `<span>` items that are not focusable, not keyboard-operable, and aren't connected to a `tabpanel`. These are progress indicators, not tabs.
- **Recommendation**: Same as M2 — use `role="list"` + `aria-current="step"`. Reserve `tablist` semantics for actual tab UIs (AssetCatalogPicker has a real one).
- **Command**: `/harden`.

**M4 · CreativeCard tile menu has no Escape-to-close**
- **Location**: `src/components/campaigns/CreativeCard.tsx:511-606`
- **Category**: Accessibility
- **Description**: Click-outside (mousedown) closes. No `Escape` keydown. No focus trap. Once opened with keyboard (Enter on the trigger), there's no way to back out short of tabbing through.
- **Recommendation**: Add Escape handler that closes + returns focus to the trigger. The Modal primitive already has this pattern — copy it.
- **Command**: `/harden`.

**M5 · BrandEditor swatch click both "removes" and X-button removes — confusing**
- **Location**: `src/components/brand/BrandEditor.tsx:211-240`
- **Category**: UX
- **Description**: Clicking the swatch itself calls `removeColor(c)`. There's also a tiny X button overlaid that does the same thing. The X is opacity-0 until hover. Touch users can't see the X. The `title="… · tap to remove"` is the only signal — and `title` doesn't show on touch.
- **Recommendation**: Pick one interaction. If the swatch click is "select" (for palette reordering or something), make the X the only delete; if the swatch IS delete, remove the X and rely on a clearer affordance (a "remove all" action elsewhere).
- **Command**: `/clarify`.

**M6 · FontPicker injects arbitrary Google Fonts via runtime `<link>` injection**
- **Location**: `src/components/onboarding/DnaStep.tsx:293-356`
- **Category**: Security (low), Performance, UX
- **Description**: User-typed font name → `https://fonts.googleapis.com/css2?family=…` link element appended to head. Input is URL-encoded so injection risk is minimal, but: (a) every keystroke commits on blur and triggers a network request; (b) failed font names silently fall through to system; (c) loaded font links accumulate across the session.
- **Recommendation**: At minimum debounce + dedupe the load. Better: ship a curated allow-list of "common brand fonts" + free-text override behind an "advanced" toggle.
- **Command**: `/harden`.

**M7 · Touch targets under 44×44px on action menus and steppers**
- **Location**: `CreativeCard.tsx:553` (`size-7` = 28×28 menu trigger), `PhotoshootWizard.tsx:572-589` (variants stepper, `h-8` = 32 in pill), `CreativeCard.tsx:292,307` (regenerate/download `h-7 w-7`)
- **Category**: Accessibility (mobile)
- **Impact**: Hard to hit on touch devices, especially for the menu trigger which lives in a corner of an image.
- **Recommendation**: Bump to `h-9 w-9` minimum (36px) or `h-11 w-11` (44px, WCAG 2.5.5 AAA).
- **Command**: `/harden`.

**M8 · CivitaiSsoButton rebuilds Button styling inline**
- **Location**: `src/components/login/CivitaiSsoButton.tsx:6-32`
- **Category**: Architecture, theming
- **Description**: Hand-rolled button with classes from the `Button` primitive duplicated. Misses the primitive's `focus-visible:ring-2 focus-visible:ring-volt` ring.
- **Recommendation**: Use `<Button variant="primary" size="lg">` and pass the custom hex icon as `leadingIcon`. Keep the `<form method="post">` wrapper.
- **Command**: `/normalize`.

**M9 · CatalogGrid filter chips don't filter**
- **Location**: `src/components/catalog/CatalogGrid.tsx:30-34`
- **Category**: UX
- **Description**: "all · N", "live · N", "archived · N" look like filter affordances. They aren't wired to any filter state.
- **Recommendation**: Either wire the filter (URL `?status=live`) or convert to `<Badge>` so they read as informational.
- **Command**: `/clarify`.

## Patterns & Systemic Issues

| Pattern | Where it bites | Root cause |
|---|---|---|
| **Interactive role on `<span>`** | `Chip` primitive; rippled through PhotoshootWizard, Catalog, AssetUploader | Primitive owns the wrong element. Fix at the source. |
| **Inputs without `<label htmlFor>`** | Onboarding's InputStep, DnaStep | `Card`/`DnaCard` use `<h4>` or eyebrow `<span>` as visual label; no programmatic link. CampaignWizard does this right — copy the pattern. |
| **Hardcoded `rgba(0,255,157,...)` & `#0a0a0f`** | AssetsGallery, AssetUploader, AddProductForm, Sidebar top-up button, BuzzGlyph | Three slightly-different "soft volt" values inline. Tokens exist; they got bypassed when an opacity tweak was needed. Need `--volt-soft` and `--volt-soft-strong`. |
| **Centered-column composition** | LoginScreen, OnboardingFrame, all onboarding steps, CampaignsList landing, CatalogGrid empty | Reflex for "make it look intentional." Three different screens, same skeleton. |
| **Same eyebrow recipe everywhere** | 80+ uses of `font-mono text-[10.5px] uppercase tracking-[0.12em] text-fg-3` | No semantic separation between "step indicator", "section label", "metadata caption". Type system says all metadata looks the same. |

## Positive Findings

**Token system is unusually thorough** — full type scale with opsz axis, articulated bloom shadow vocabulary, light theme already worked out (even if no switcher exists yet), motion tokens with proper easing names. Don't fragment it.

**CampaignWizard is the reference architecture** — URL-as-state for steps, debounced previews via `useCampaignPreview`, proper `htmlFor`/`id` labeling, `role="status" aria-live="polite"` on the drafting overlay, `aria-pressed`/`aria-expanded` on toggle states. Use it as the template for the photoshoot wizard cleanup.

**AssetCatalogPicker** — `role="tablist"` correctly applied with `aria-selected`, `role="listbox"`/`role="option"` with `aria-multiselectable`, `aria-disabled` where appropriate. This is what good ARIA looks like in this codebase.

**Wordmark** — distinctive, well-executed; the gradient on the "v" is the one place gradient text actually carries identity rather than decoration.

**CreativeCard polling + skeleton placeholders** — `data-testid="image-skeleton"` slots, status badges with semantic dots, real download-as-zip for multi-image tiles. Genuinely good interaction design.

**`AssetTile` keyboard story** — `aria-label`, `focus-visible:ring-2 focus-visible:ring-volt`, `loading="lazy"`. This is the one card-grid in the codebase that's done right.
